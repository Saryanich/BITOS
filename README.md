# 🖥️ BITOS — Эмулятор Операционной Системы на Python

BITOS (v05V6) — это интерактивная графическая среда, имитирующая полноценную операционную систему. Проект построен на стандартной библиотеке Tkinter с использованием современных принципов UI/UX, анимированных эффектов, многопоточности и продвинутой системы логирования событий и безопасности.

---

## ✨ Ключевые Особенности

### 🚀 1. Интерактивный SplashScreen
* Плавный запуск: Экран загрузки со стильным кастомным градиентом.
* Частицы и анимация: Динамический рендеринг звездного неба/частиц с помощью Tkinter Canvas.
* Отслеживание прогресса: Визуальный индикатор (ProgressBar) со статус-текстом, отображающим этапы инициализации подсистем.

### 🔒 2. Безопасность и Авторизация (LoginScreen)
* Защита от Брутфорса: Ограничение количества попыток ввода (максимум 5). При превышении лимита система временно блокируется на 60 секунд.
* Хеширование паролей: Безопасное хранение PIN-кодов с использованием криптографического алгоритма SHA-256.
* Автономность: Наличие встроенного диалогового окна для быстрой смены текущего PIN-кода пользователем.
* Управление питанием: Возможность выключения и перезагрузки хост-системы Windows прямо с экрана блокировки.

### 📂 3. Полноценный Рабочий Стол и Проводник
* Менеджер ярлыков: Размещение иконок по сетке, поддержка Drag-and-Drop, контекстные меню для каждого ярлыка.
* Встроенный Проводник (Explorer): Навигация, создание файлов (.txt, .html, .py, .js, .json) и папок.
* Умная Корзина (TrashManager): Перемещение файлов в изолированную временную директорию с возможностью восстановления или безвозвратного удаления.

### 🔔 4. Системный Трей и Уведомления
* Центр уведомлений: Всплывающие уведомления (Toast notification) с анимацией, счетчиком непрочитанных сообщений и историей.
* Регулятор Громкости: Кастомный микшер громкости BITOS с динамической сменой иконок (🔇, 🔈, 🔉, 🔊).

### 📌 5. Кастомизация и Виджеты
* Виджеты на Canvas: Возможность выноса на рабочий стол интерактивных часов, календаря и текстовых стикеров-напоминаний.
* Динамические Темы: Полная поддержка кастомизации интерфейса и налету изменяющихся цветовых щем для всех системных окон.

### 📝 6. Аудит и Логирование (SecurityManager)
Система ведет раздельный учёт всех действий в сессии для обеспечения максимальной прозрачности:
* access.log — входы, выходы и попытки авторизации.
* files.log — чтение, создание, перемещение и удаление файлов.
* applications.log — история запуска приложений и создание ярлыков.
* security.log — фиксация критических событий (попытки брутфорса, смена PIN).
* error.log — трассировка непредвиденных ошибок модулей.

## 💻 Системные Требования

* ОС: Windows 7 и новее (для корректной работы системных вызовов).
* ОЗУ: от 4 ГБ (рекомендуется 8 ГБ).
* Диск: 35 МБ свободного места.
* Важно: Для полноценного доступа к файловому менеджеру рекомендуется использовать внешний накопитель (флешку), так как прямой просмотр путей диска C:\ может быть ограничен в целях безопасности хост-системы.

## 🛠️ Установка зависимостей

Большая часть функционала BITOS работает на стандартной библиотеке Python, однако для работы расширенных модулей (перехват клавиш, галерея, WinAPI) необходимо установить дополнительные библиотеки.

Для установки выполните следующие команды в терминале:

py -m pip install keyboard
py -m pip install Pillow
py -m pip install pywin32
py -m pip install requests

> Примечание:
> * keyboard необходим для продвинутой блокировки клавиш.
> * Pillow активирует графическую галерею и работу с изображениями.
> * pywin32 используется подсистемой для автоматического определения подключенных USB-накопителей.

## ⚙️ Первое включение и Вход

При первом запуске BITOS автоматически сгенерирует необходимую структуру папок, файлы конфигурации безопасности, журналы логов и создаст персональную лицензию. Первая инициализация может занять от 5 до 60 секунд.

* Стандартный Логин: User
* Стандартный PIN-код: 1234
* Режим разработчика: Пароль BIT642Os

## 📁 Структура создаваемых директорий

После запуска скрипт разворачивает в своей директории следующую древовидную структуру:

📂 BITOS Project
│
├── 📄 05V6 (6).pyw           # Основной исполняемый файл системы
│
└── 📂 System
    ├── 📂 Security           # Хеши PIN-кодов и конфигурация доступа
    ├── 📂 Temp
    │   └── 📂 Trash          # Изолированная папка корзины
    └── 📂 Logs               # Журналы логирования подсистем
        ├── 📄 access.log
        ├── 📄 files.log
        ├── 📄 security.log
        └── 📄 error.log

## 💡 Использование

Запустите главный файл скрипта с помощью интерпретатора Python:

python "05V6 (6).pyw"

Для предотвращения появления консольного окна на заднем плане в операционной системе Windows, расширение файла по умолчанию установлено как .pyw, что автоматически запускает его через pythonw.exe.
